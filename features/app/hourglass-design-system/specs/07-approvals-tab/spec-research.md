# Spec Research: 07-approvals-tab

## Problem Context

`app/(tabs)/approvals.tsx` is the manager view for approving/rejecting manual time entries
and overtime requests. It currently uses `StyleSheet.create()` + hardcoded hex. The rebuild
needs to apply design tokens, update `ApprovalCard` to use `className`, and ensure the
swipe-to-approve/reject gesture UX remains intact.

This is a manager-only screen — contributors see a redirect message. Both states need design
system treatment.

## Exploration Findings

**Current approvals.tsx structure:**
- Role guard: if not manager, show "You're a contributor" message
- Header: "Approvals" + count badge
- FlatList of `ApprovalCard` items (manual time + overtime)
- Bottom: "Approve All" bulk action button
- `ApprovalCard` uses `Animated` + `PanResponder` for swipe gestures
- `RejectionSheet` bottom sheet for rejection reason input
- Data from `useApprovalItems()` hook

**ApprovalCard current implementation:**
- `PanResponder` + core `Animated` API (NOT Reanimated)
- Design rule: "always Reanimated v4" — but gesture handling with Reanimated requires
  `react-native-gesture-handler` + `useAnimatedGestureHandler` or Gesture API
- Decision: **Keep `PanResponder` + core Animated for the gesture layer** (migration to
  Reanimated Gesture API is significant scope; correctness > purity for gestures in this spec)
- BUT: all visual styling (colors, radius, spacing) MUST use NativeWind className

**The "no core Animated API" rule from gemini-ui.ts applies to GENERATED components:**
The brand guideline is for new components generated by AI tools. For existing complex gesture
components being migrated, keeping PanResponder is pragmatic. Note the exception in comments.

**Data from `useApprovalItems()`:**
```typescript
interface ApprovalItem {
  id: string;
  type: 'manual' | 'overtime';
  employeeName: string;
  hours: number;            // Hours submitted
  date: string;             // YYYY-MM-DD or week start
  cost: number;             // Calculated cost ($)
  status: 'PENDING';
  memo?: string;            // Contributor's description
}

// Returns:
interface UseApprovalItemsResult {
  items: ApprovalItem[];
  isLoading: boolean;
  error: Error | null;
  approve: (id: string) => Promise<void>;
  reject: (id: string, reason: string) => Promise<void>;
  approveAll: () => Promise<void>;
}
```

**Empty state (manager, no pending approvals):**
Currently shows earnings-focused view — good UX pattern to keep, just update styling.

**Contributor empty state:**
Simple message + icon. Update to dark glass aesthetic.

**RejectionSheet:**
Bottom sheet (Modal) for entering rejection reason. Update to use design tokens.

## Key Decisions

1. **ApprovalCard gesture**: Keep `PanResponder` + core `Animated` API for gesture mechanics.
   Update ALL visual styles (colors, backgrounds, borders, typography) to NativeWind className.
   Add code comment: "// Gesture: PanResponder retained — Reanimated gesture migration is out of scope"

2. **Card layout**: Each approval card uses the `Card` component (from spec 03) as wrapper.
   Swipe content (approve: success green, reject: destructive red) added as absolute-positioned
   background behind the card content.

3. **Type badges**: Manual time = gold pill, Overtime = warning pill.
   Using `bg-gold/20 text-gold rounded-full` and `bg-warning/20 text-warning rounded-full`.

4. **Bulk approve**: `Pressable` with `bg-success rounded-xl py-4` — full-width bottom button.

5. **Manager empty state (no pending)**: Shows a simple Card with a MetricValue of weekly
   pending count (0) and a message "All clear — no pending approvals".

6. **Contributor redirect**: View with bg-surface rounded-2xl, text-textSecondary message,
   icon placeholder.
