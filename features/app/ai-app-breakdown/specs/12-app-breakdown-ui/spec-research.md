# Spec Research: 12-app-breakdown-ui

## Problem Context

With 12 weeks of per-app AI vs non-AI slot data now stored (from spec 11-app-data-layer), this spec builds the UI that surfaces it. The key framing: WorkSmart auto-classifies AI usage — users cannot manually tag. The goal is to show *which apps the system doesn't credit as AI* so the user knows where to do more AI-assisted work to increase their score.

## Exploration Findings

### ai.tsx card order (app/(tabs)/ai.tsx)
Current cards, top to bottom:
1. AIArcHero (ring + BrainLift progress bar)
2. Prime Radiant (AIConeChart)
3. Daily Breakdown (DailyAIRow list)
4. 12-Week Trajectory (TrendSparkline + tier legend)
5. How it's calculated (legend)
6. Last fetched timestamp

New "App Breakdown" card inserts between **Daily Breakdown (3)** and **12-Week Trajectory (4)** — logical flow: today's detail → this week's apps → long-term trend.

### Stagger count (ai.tsx:63)
`useStaggeredEntry({ count: 6 })`. Adding one card → bump to 7. The new card uses `getEntryStyle(3)`, and Trajectory/Legend/Timestamp shift to 4/5/6.

### Card component pattern (src/components/Card.tsx)
```tsx
<Card borderAccentColor={colors.cyan}>
  <SectionLabel className="mb-3">LABEL</SectionLabel>
  {/* content */}
</Card>
```
App Breakdown card follows this pattern with `borderAccentColor={colors.violet}` (AI purple = app theme).

### DailyAIRow pattern (src/components/DailyAIRow.tsx)
Per-row component with date label + metrics. App breakdown rows follow a similar flat list pattern but with app name + two-segment bar instead of date + percentages.

### Colors available (src/lib/colors.ts)
- `colors.cyan` — AI-credited (positive signal)
- `colors.border` — non-AI / empty
- `colors.violet` — AI theme accent (used for card border)
- `colors.textPrimary`, `colors.textSecondary`, `colors.textMuted`

### Existing two-bar component
`ProgressBar` (`src/components/ProgressBar.tsx`) is single-segment. App breakdown needs a **two-segment** bar (AI | non-AI), so a new `AppUsageBar` component is needed.

## Key Decisions

1. **Two-segment bar, not stacked** — Show AI (cyan) and non-AI (border/grey) segments side by side proportional to their slot counts. Width = (aiSlots / totalSlots) × 100% for cyan. Simple to read.

2. **Top 8 apps max** — Cap list at 8 entries to keep the card scannable. Sort by total slots (aiSlots + nonAiSlots) descending — highest-usage apps first.

3. **Guidance chips below the list** — 1–3 chips generated by `generateGuidance()`. Each chip is a short sentence with a color dot. No taps, no navigation — purely informational in v1.

4. **Aggregate 12w, not current week only** — Use `aggregated12w` from `useAppBreakdown` as the primary view. Current week alone has too few slots on Monday/Tuesday to be meaningful. 12-week aggregate gives a stable signal.

5. **Guidance tone: opportunity-focused** — "Chrome is your top untagged app — using AI tools there could increase your score" not "you should tag Chrome." WorkSmart auto-classifies; tagging is not in the user's control. The action is to use AI tools within those apps.

6. **Minimum slot threshold** — Only show apps with ≥3 total slots in the 12-week aggregate to filter out transient apps. 3 slots = 30 minutes minimum presence.

## Interface Contracts

### AppUsageBar (NEW — src/components/AppUsageBar.tsx)
```typescript
interface AppUsageBarProps {
  aiSlots: number;          // AI-only (ai_usage but not second_brain)
  brainliftSlots: number;   // second_brain slots (shown as violet segment)
  nonAiSlots: number;
  height?: number;           // default: 4
  className?: string;
}
export default function AppUsageBar(props: AppUsageBarProps): JSX.Element
```
- Renders a **three-segment** View:
  - Violet (brainliftSlots): `colors.violet`
  - Cyan (aiSlots - brainliftSlots, i.e. AI-only): `colors.cyan`
  - Grey (nonAiSlots): `colors.border`
- Uses flex proportions for all three segments
- If brainliftSlots=0 → violet segment omitted (just cyan + grey)
- If all zero → renders empty bar (full border-color)
- No animation (static)
- Source: aiSlots, brainliftSlots, nonAiSlots from AppBreakdownEntry (← spec 11)

Note: `aiOnlySlots = aiSlots - brainliftSlots` (since brainliftSlots ⊆ aiSlots)

### GuidanceChip (NEW — src/lib/appGuidance.ts)
```typescript
export interface GuidanceChip {
  text: string;   // Short sentence, max ~60 chars
  color: string;  // hex color for dot indicator
}

export function generateGuidance(
  aggregated: AppBreakdownEntry[],
  currentWeek: AppBreakdownEntry[],
): GuidanceChip[]
```
Source of aggregated/currentWeek: ← `useAppBreakdown` (← spec 11)

**Guidance rules (evaluated in order, max 3 chips returned):**

Rule 1 — Top opportunity:
- Find app with highest `nonAiSlots` AND `nonAiSlots / (aiSlots + nonAiSlots) > 0.5`
- Chip: `"${appName} is your top untagged app — try using AI tools there"`
- Color: `colors.warning`

Rule 2 — AI leader app:
- Find app with `aiSlots / (aiSlots + nonAiSlots) >= 0.8` AND aiSlots ≥ 5
- Chip: `"${appName} is your strongest AI app — ${Math.round(pct)}% AI-credited"`
- Color: `colors.cyan`

Rule 3 — BrainLift app highlight:
- Find app with highest `brainliftSlots` AND brainliftSlots ≥ 3
- Chip: `"${appName} drives most of your BrainLift time — keep it up"`
- Color: `colors.violet`

Rule 4 — AI progress this week:
- If `currentWeek` total AI slots > 0: compute AI% for current week vs 12w aggregate
- If current week AI% > aggregate AI% + 5 → `"You're above your 12-week average this week"`
- Color: `colors.success`
- If current week AI% < aggregate AI% - 5 → `"Slower AI week — still time to close the gap"`
- Color: `colors.warning`

## Test Plan

### AppUsageBar
**Signature:** `({ aiSlots, brainliftSlots, nonAiSlots, height?, className? }) → JSX.Element`

**Happy Path:**
- [ ] aiSlots=60, brainliftSlots=30, nonAiSlots=10 → three segments with correct flex (violet=30, cyan=30, grey=10)
- [ ] brainliftSlots=0 → no violet segment, just cyan + grey
- [ ] All non-AI (aiSlots=0, brainliftSlots=0) → single grey segment only

**Edge Cases:**
- [ ] All zero → renders empty bar (border-color full width)
- [ ] brainliftSlots > aiSlots → clamp aiOnly to 0 (defensive)
- [ ] height prop overrides default 4

**Mocks Needed:** None (pure rendering)

### generateGuidance
**Signature:** `(aggregated, currentWeek) → GuidanceChip[]`

**Happy Path:**
- [ ] Top untagged app with >50% nonAi → returns opportunity chip naming the app
- [ ] App with ≥80% AI and ≥5 slots → returns leader chip
- [ ] App with highest brainliftSlots ≥ 3 → returns BrainLift chip (violet)
- [ ] Current week AI% > 12w avg + 5 → returns progress chip
- [ ] Returns max 3 chips

**Edge Cases:**
- [ ] Empty aggregated → returns `[]`
- [ ] No app qualifies any rule → returns `[]`
- [ ] App barely over threshold (50.1% non-AI) → still triggers rule 1
- [ ] currentWeek empty → rule 3 not evaluated

**Mocks Needed:** None (pure function)

### AppBreakdownCard
**Happy Path:**
- [ ] Renders card with SECTION LABEL "APP BREAKDOWN"
- [ ] Shows entries up to 8 rows
- [ ] Each row shows appName, AppUsageBar, slot count
- [ ] Guidance chips render below list
- [ ] Returns null when entries=[]

**Edge Cases:**
- [ ] entries.length > 8 → only first 8 shown (caller responsibility, but card handles gracefully)
- [ ] guidance=[] → no guidance section rendered (no empty space)

**Mocks Needed:**
- `useAppBreakdown`: mock to return test data

### ai.tsx integration
- [ ] AppBreakdownCard renders between Daily Breakdown and Trajectory when data present
- [ ] Card hidden when aggregated12w is empty
- [ ] Stagger count is 7 (not 6)
